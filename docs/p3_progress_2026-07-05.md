# P3 进度纪要 2026-07-05:时序生成预研(主线 B 首轮)

> 对应 [work_plan.md](work_plan.md) P3(计划 09-03 开工,**提前约两个月,以"并行预研"名义启动**;G2 正式判定挂起待 VoD 下游实验)
> 数据:mini sweeps(~20Hz)时序配对 **(t, t+5)≈0.25s**,11,246 对(`temporal_mini_k5`,双条件云)

## 1. 实验设计

上一帧条件的下一帧扩散生成,唯一变量是**条件云的构造**:
- `ts_ego`:上一帧点仅按自车位姿刚体变换到目标帧(ego-warp,稳健下限)
- `ts_dopp`:上一帧点先做**门控 Doppler 径向推进**(|残差|>1 的点按 `res·Δt·r̂` 前推)再 ego 变换(B 线核心机制)

其余全同(骨架/种子/40k 步/CFG/EMA/静态物理损失 λ=0.1/ego 条件)。新增**"复制条件"平凡基线**:直接拿条件云当预测,量出时序条件的信息量上界参照。

## 3. 三个结论(两个如实的负/中性结果 + 一个方向)

1. **时序条件 ≫ LiDAR 条件**(全指标)——上一帧雷达云是比 LiDAR 强得多的条件信号,B 线的数据面价值确认。
2. **[否定结果] dopp-warp 条件 vs ego-warp 条件在 0.25s 视距无生成增益**:两臂指标全部在噪声级内。0.25s 内门控动态点径向位移(中位~0.5m)相对雷达稀疏度太小,网络从两种条件里提取的信息等价。**Doppler-warp 的价值必须到更长视距/多步 rollout 才能兑现**(07-04 的 warp 验证在 0.5s 已见分离)——下一轮用 K=10 或多步。
3. **[关键发现] 复制条件基线(CD 2.94)完爆生成(7.15)**:相邻帧高度相似,而从纯噪声起步的扩散远未吃满条件信息。**架构启示:时序生成应以 warp 后的条件云为"草稿"做部分扩散/残差精修**(SDEdit 式:从条件云加噪到中间步再去噪,而非从 x_T 从头生成)——这与 proposal"Doppler 驱动帧间 warp 为骨架、生成补 innovation"的设计完全一致,现在有了定量依据。

## 4. 第二轮:SDEdit 部分扩散 + K=10 长视距(同日)

### 4.1 SDEdit 扫描(纯推理,t_start ∈ {0,200,400,600,1000})——负结果,定位模型瓶颈

K=5 与 K=10 上结论一致:**任何程度的"精修"都损伤草稿**(K=5 ego:复制 CD 2.94 → t_start=200 就崩到 7.06 ≈ 从头 7.15)。当前 6.6M 小模型的去噪是把点云往自己偏模糊的分布上拉,而非"保留草稿+修正差异"。W1/动态占比同样复制最优——0.25s 帧间差本就接近不可约底线,生成无施展空间。

### 4.2 K=10(0.5s)——**方向性信号:dopp-warp 草稿开始占优(视距翻转)**

| 复制条件基线 CD | K=5(0.25s) | K=10(0.5s) |
|---|---|---|
| ego-warp 草稿 | **2.940** | 3.493 |
| dopp-warp 草稿 | 3.172 | **3.451** |

**0.25s 时 dopp 反而差(门控推进对杂波的放大 > 对动态点的修正);0.5s 时翻转为优**(CD −1.2%,CD_Doppler 3.757 vs 3.796)——与 07-04 关键帧验证(0.5s 动态点 NN 1.80 vs 1.98)方向一致。全云 CD 被 ~97% 静态点稀释,效应小但稳定;**warp 的价值随视距增长,与 B 线"长时距时序一致性"的定位吻合**。K=10 训练臂(ts_ego10/ts_dopp10)生成指标仍互相打平且劣于复制(7.7-7.9 vs 3.45)。

### 4.3 本轮总结论 → 架构定型依据

三条证据链(从头生成 ≪ 复制;SDEdit 精修损伤草稿;dopp 草稿 0.5s 起占优)共同指向:**B 线的正确架构不是"条件扩散"也不是"SDEdit 后精修",而是把 draft→GT 学成传输问题——桥式扩散(I2SB/flow-matching 风格):训练时噪声轨迹从 warp 草稿分布出发到目标帧分布**,模型显式学习"草稿的残差修正",天然保草稿、天然利用 dopp-warp 的更优起点,`L_temp` 也在此框架内自然表达。这是下一轮的主攻方向。

**前置条件(诚实)**:单帧生成质量(模型规模/点数/数据量)是所有时序结论的地板——trainval(522G)存储审批越来越关键。

## 5. 第三轮:桥式扩散原型 → **架构验证成功,生成臂首次逼近复制且修复其物理陈旧**

**实现**([train_bridge.py](../code/scripts/train_bridge.py)):rectified-flow 式 draft→GT 传输。离线 OT 配对([build_bridge_perm.py](../code/scripts/build_bridge_perm.py),匈牙利指派 11,101 对/130s)解决点集无对应;训练 `x_t=(1−t)·draft_π+t·GT+σ(1−t)ε`,模型学位移场 `v≈GT−draft_π`;推理从草稿 Euler 积分 50 步(**无需配对**);静态物理约束加在 `x̂1=x_t+(1−t)v̂` 上。复制基线即 v≡0 特例。

| val N=24 中位(K=10) | 条件扩散 | **bridge_ego** | **bridge_dopp** | 复制 | GT |
|---|---|---|---|---|---|
| CD | 7.66-7.87 | 4.050 | **4.004** | **3.45-3.49** | (锚11.1) |
| **MMD** | 0.047-0.049 | **0.0100(↓79%)** | 0.0101 | — | |
| **JSD** | 0.311-0.327 | **0.169(↓46%)** | 0.175 | — | |
| W1(v_r) | 1.90-1.93 | 1.109 | **1.005** | 0.314 | 0 |
| 动态样占比 | 24% | 43.2% | **45.0%** | 63.3% | 57.7% |
| **PCE<0.5** | 63-66%(过度静化) | **40.4%** | 35.8% | **19.7%(陈旧!)** | **39.0%** |

**四个结论**:
1. **架构判断验证**:CD 7.7→4.0(近腰斩),分布指标(MMD/JSD)成倍优于条件扩散——"draft→GT 学传输"是正确形态。BEV 上桥式输出**完整保留道路线性结构、覆盖全距离**(条件扩散的弥散/方盒问题消失)。
2. **桥式修复复制的"物理陈旧"**:复制虽 CD 最低(3.45),但其 PCE<0.5 仅 **19.7%**——warp 过来的旧点不满足新帧物理;桥式把它修回 **40.4% ≈ GT 39.0%**。叙事完整:**复制几何近但物理旧,桥式以微小几何代价换物理/分布真实**——这正是"warp 骨架 + 生成补差"的定量兑现。
3. dopp vs ego 草稿:dopp 在 Doppler 相关指标一致占优(CD 4.004<4.050、W1 1.005<1.109、动态样 45.0>43.2),幅度小——与草稿差(3.451 vs 3.493)同量级传导,机制自洽。
4. 待查项:个别 val 样本生成的 v_r 符号/色相与 GT 不符(样本图 #0),需 spot-check v_r 在桥式传输中的符号处理。

## 6. 第四轮:v_r spot-check + 多步 rollout 漂移测试(G3 预演)

**spot-check 结论:无 bug**——8 个抽检配对 draft/GT 的 v_r 符号一致率 90–100%、中位数吻合;样本图"色相翻转"是子图各自动色标的视觉假象(后续作图统一 `vmin/vmax=±15`)。

**rollout(K=10≈0.5s/步 × 5 步 = 2.5s,12 segments,四臂自回归)**([eval_rollout.py](../code/scripts/eval_rollout.py)):

| t(s) | copy_ego | copy_dopp | bridge_ego | bridge_dopp | ← CD med |
|---|---|---|---|---|---|
| 0.5 | **3.63** | 3.89 | 4.56 | 4.47 | |
| 1.5 | **4.09** | 4.80 | 5.94 | 6.06 | |
| 2.5 | **5.29** | 6.15 | 7.46 | 8.26 | |

| t(s) | copy_dopp | bridge_dopp | GT | ← PCE<0.5(物理新鲜度) |
|---|---|---|---|---|
| 0.5 | 36.9% | **50.7%** | 41.6% | |
| 1.5 | 37.4% | **44.9%** | 41.7% | |
| 2.5 | **8.6%(崩)** | **25.1%** | 40.7% | |

**读数(如实)**:
1. **G3 尚未通过**:几何漂移上纯 ego-warp 链仍是下限赢家——桥式每步 ~0.5-1m 的生成税随步数复利(还有分布偏移:模型只见过真实帧草稿,rollout 喂进的是生成帧)。
2. **但物理新鲜度维度桥式大胜**:copy_dopp 的陈旧 v_r 在 2.5s 崩到 8.6%,桥式保持 25.1%(GT 40.7%);copy_dopp 几何也比 copy_ego 漂得快(6.15 vs 5.29)——**"不更新 v_r 的 warp 链会被自己的旧 Doppler 带偏"**,这正是 B 线"必须联合生成 Doppler"的核心论据,现在有了 2.5s 尺度的定量证据。
3. **通往 G3 的三个修理点**(明确、可执行):① rollout-aware 训练(草稿噪声增广/scheduled sampling,治分布偏移);② 更大模型 + trainval 全量(治每步生成税);③ `L_temp` 锚定相邻步。

## 7. 第五轮:G3 复赛(增广 + L_temp)——仍未过,但修理点排序被数据修正

**实现**:train_bridge v2——① 草稿/条件白噪增广(p=0.5,σ~U(0,1m))模拟生成帧输入;② **L_temp = |Δrange − v̄_r·Δt|**(利用桥式 OT 对应:p0=同批 prev 点纯 ego-warp 位置为锚,梯形平均 v̄_r,GT-草稿匹配距离做持久点门控)。双臂:v2aug(仅增广)/ v2full(增广+L_temp λ=0.1)。

**单步指标**:v2full 的 **W1(v_r)=0.706(历史最佳,v1 为 1.005)**、动态样 46.3%——`L_temp` 确实提升 Doppler 真实性;CD 4.63(增广税,v1 4.00)。

**rollout 复赛(2.5s)**:

| CD@2.5s | copy_ego | br_dopp(v1) | v2aug | v2full |
|---|---|---|---|---|
| | **5.27** | 7.15 | 7.65 | 8.27 |

| PCE<0.5@2.5s | copy_dopp | br_dopp(v1) | v2aug | v2full | GT |
|---|---|---|---|---|---|
| | 8.6% | **27.4%** | 14.5% | 26.3% | 39.8% |

**结论(如实)**:
1. **[证伪] 白噪增广治不了 rollout 漂移**——生成云的误差是**结构化**的(模糊、簇偏移),不是各向同性白噪;v2 两臂 rollout CD 反而更差。真正的 scheduled sampling(训练时喂真实生成帧)才是对症药,但工程成本高一档。
2. `L_temp` 价值确认:单步 Doppler 真实性最佳 + rollout 物理新鲜度在增广臂中恢复(26.3% vs v2aug 14.5%)——**保留**。
3. **G3 的主杠杆收敛为一个:规模**(更大模型 + trainval 全量数据治每步生成税)。mini 尺度上的便宜修理已挖尽——**trainval 存储审批现在是 B 线的第一阻塞项**。
