# P3 进度纪要 2026-07-05:时序生成预研(主线 B 首轮)

> 对应 [work_plan.md](work_plan.md) P3(计划 09-03 开工,**提前约两个月,以"并行预研"名义启动**;G2 正式判定挂起待 VoD 下游实验)
> 数据:mini sweeps(~20Hz)时序配对 **(t, t+5)≈0.25s**,11,246 对(`temporal_mini_k5`,双条件云)

## 1. 实验设计

上一帧条件的下一帧扩散生成,唯一变量是**条件云的构造**:
- `ts_ego`:上一帧点仅按自车位姿刚体变换到目标帧(ego-warp,稳健下限)
- `ts_dopp`:上一帧点先做**门控 Doppler 径向推进**(|残差|>1 的点按 `res·Δt·r̂` 前推)再 ego 变换(B 线核心机制)

其余全同(骨架/种子/40k 步/CFG/EMA/静态物理损失 λ=0.1/ego 条件)。新增**"复制条件"平凡基线**:直接拿条件云当预测,量出时序条件的信息量上界参照。

## 3. 三个结论(两个如实的负/中性结果 + 一个方向)

1. **时序条件 ≫ LiDAR 条件**(全指标)——上一帧雷达云是比 LiDAR 强得多的条件信号,B 线的数据面价值确认。
2. **[否定结果] dopp-warp 条件 vs ego-warp 条件在 0.25s 视距无生成增益**:两臂指标全部在噪声级内。0.25s 内门控动态点径向位移(中位~0.5m)相对雷达稀疏度太小,网络从两种条件里提取的信息等价。**Doppler-warp 的价值必须到更长视距/多步 rollout 才能兑现**(07-04 的 warp 验证在 0.5s 已见分离)——下一轮用 K=10 或多步。
3. **[关键发现] 复制条件基线(CD 2.94)完爆生成(7.15)**:相邻帧高度相似,而从纯噪声起步的扩散远未吃满条件信息。**架构启示:时序生成应以 warp 后的条件云为"草稿"做部分扩散/残差精修**(SDEdit 式:从条件云加噪到中间步再去噪,而非从 x_T 从头生成)——这与 proposal"Doppler 驱动帧间 warp 为骨架、生成补 innovation"的设计完全一致,现在有了定量依据。

## 4. 第二轮:SDEdit 部分扩散 + K=10 长视距(同日)

### 4.1 SDEdit 扫描(纯推理,t_start ∈ {0,200,400,600,1000})——负结果,定位模型瓶颈

K=5 与 K=10 上结论一致:**任何程度的"精修"都损伤草稿**(K=5 ego:复制 CD 2.94 → t_start=200 就崩到 7.06 ≈ 从头 7.15)。当前 6.6M 小模型的去噪是把点云往自己偏模糊的分布上拉,而非"保留草稿+修正差异"。W1/动态占比同样复制最优——0.25s 帧间差本就接近不可约底线,生成无施展空间。

### 4.2 K=10(0.5s)——**方向性信号:dopp-warp 草稿开始占优(视距翻转)**

| 复制条件基线 CD | K=5(0.25s) | K=10(0.5s) |
|---|---|---|
| ego-warp 草稿 | **2.940** | 3.493 |
| dopp-warp 草稿 | 3.172 | **3.451** |

**0.25s 时 dopp 反而差(门控推进对杂波的放大 > 对动态点的修正);0.5s 时翻转为优**(CD −1.2%,CD_Doppler 3.757 vs 3.796)——与 07-04 关键帧验证(0.5s 动态点 NN 1.80 vs 1.98)方向一致。全云 CD 被 ~97% 静态点稀释,效应小但稳定;**warp 的价值随视距增长,与 B 线"长时距时序一致性"的定位吻合**。K=10 训练臂(ts_ego10/ts_dopp10)生成指标仍互相打平且劣于复制(7.7-7.9 vs 3.45)。

### 4.3 本轮总结论 → 架构定型依据

三条证据链(从头生成 ≪ 复制;SDEdit 精修损伤草稿;dopp 草稿 0.5s 起占优)共同指向:**B 线的正确架构不是"条件扩散"也不是"SDEdit 后精修",而是把 draft→GT 学成传输问题——桥式扩散(I2SB/flow-matching 风格):训练时噪声轨迹从 warp 草稿分布出发到目标帧分布**,模型显式学习"草稿的残差修正",天然保草稿、天然利用 dopp-warp 的更优起点,`L_temp` 也在此框架内自然表达。这是下一轮的主攻方向。

**前置条件(诚实)**:单帧生成质量(模型规模/点数/数据量)是所有时序结论的地板——trainval(522G)存储审批越来越关键。

## 5. 下一轮(优先级排序)
