残差图(三类均为 0 处尖峰):[assets/static_dynamic_validation.png](assets/static_dynamic_validation.png) · 完整输出:[assets/static_dynamic_validation.txt](assets/static_dynamic_validation.txt)

**读数**:
1. **解析硬约束成立且很紧**:强反射、点-面关联干净的框内静止目标 MAD 仅 0.092 m/s、95% 内点——静态 Doppler 可解析给定的前提坐实;背景较松(75.1%@0.5)是杂波/多径的贡献,恰是"生成时需要物理先验去噪"的证据。
2. **ω×r 修正有实测收益**(内点率@0.25 提升 4.3pp、MAD −12%)——训练管线保留该项。
3. **动态软约束可行但必须"软"**:框速度为差分近似(噪声)+ 刚体近似,67.4%@0.5 支持 proposal 的 `L_dop` 用稳健核/软加权而非硬约束。
4. **点云构成**:背景 94.2% / 框内静止 2.1% / 框内运动 3.7%;运动类别以 car/truck/trailer 为主。

## 3. 发现的数据陷阱(R3 风险实证,写论文可用)

`box_velocity`(标注框差分)对**真静止物**也会给出 >0.5 m/s 的伪速度——运动类别 Top8 里混入 `traffic_sign`(12,399 点)与 `trafficcone`,即**标注抖动会污染静/动分割**。处置(P2 落实):静/动判定改用 attribute 或提高阈值 + 双门限;约束按 proposal 设软加权。这正是风险 R3 预判的"分割噪声",现在有了量化证据。
