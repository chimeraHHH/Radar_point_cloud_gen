# P2 进度纪要 2026-07-05:可微物理约束 + 消融(主线 A 首轮)

> 对应 [work_plan.md](work_plan.md) P2(计划 08-06 开工,**提前约一个月**)· 基线对照见 [p1_progress_2026-07-04.md](p1_progress_2026-07-04.md) §6

## 1. 可微物理约束设计([losses/physics.py](../code/losses/physics.py))

静态解析关系 `v_r = -v_plat(p)·r̂`,`v_plat(p) = v_ego_s + ω_s×(p + t_s)`(任意点可算,叉积恒等式化到传感器系,与逐点缓存值核对 Δ<1e-6)。三个设计要点:

1. **自门控**:`w = exp(-r²/2τ²)`(r detach)——只把已接近静态流形的点拉到严格一致,真动态点/杂波权重≈0。**无需运动分割标注**,直接吸收 07-04 时序预演"未门控推进放大杂波"的教训。
2. **x̂0 桥接**:约束施加在扩散预测的干净样本 x̂0(反归一化到物理单位)上,耦合生成的几何(→r̂)与生成的 v_r。
3. **ᾱ_t 加权**:高噪声步 x̂0 不可靠时约束自动趋零。

配对缓存升级 v2(存 `v_ego_s/ω_s/t_s`);新指标 **PCE**(物理一致性误差):生成云对静态解析关系的残差统计,与同场景 GT 云的 PCE 对照解读。

## 2. 消融轮 1:λ=0 vs λ=0.1(无 ego 条件)

同种子/数据/协议,双 L40S 并行 40k 步(~25 分钟):

| val(N=24 中位) | base(λ=0) | phys(λ=0.1) | GT 锚 |
|---|---|---|---|
| **PCE med\|r\|** | 8.06 | **6.98(−13%)** | 2.25 |
| **v_r std** | 4.99 | **4.65** | 3.52 |
| CD | **8.08** | 8.34 | (场景间 12.23) |
| CD_Doppler | 11.97 | 12.21 | |
| JSD | 0.370 | 0.405 | |

**读数**:物理损失方向有效(PCE↓13%、v_r 分布向 GT 收),但带轻微质量税;两臂的绝对 PCE 都远差于 GT(约 7-8 vs 2.25 m/s)。

**关键设计发现(本轮最大价值)**:模型**没有 ego 运动条件输入**,只能从 LiDAR 隐式猜 v_ego → 推理时生成的 v_r 原理上不可能与真实 `-v_plat·r̂` 自洽,物理损失只能隔靴搔痒。**v_ego 是部署时永远可得的廉价条件**(4D-RaDiff 同样用 ego 信息),必须显式输入。

> 注:GT 锚 PCE(med 2.25 / <0.5 仅 36.4%)显著差于 07-04 全 mini 统计(75%@0.5)——val 两场景为**雨天高速+城市**,杂波/动态占比高,且无低速帧过滤;PCE 必须配对同场景 GT 锚解读,已写进指标文档。

## 3. 消融轮 2:+ego 条件 × ±物理损失 → **ego 条件是决定性因素;静态-only 约束暴露动态塌缩**

模型新增 `ego_mlp`(9 维 ego 运动→全局条件,不参与 CFG drop)。四臂全景(同种子/数据/协议):

| val N=24 中位 | base | phys | **ego_base** | **ego_phys** | GT 锚 |
|---|---|---|---|---|---|
| **PCE med\|r\|** | 8.06 | 6.98 | **0.92(−89%)** | **0.148** | 2.25 |
| PCE <0.5 m/s | 2.9% | 1.3% | 28.6% | **89.3%** | 36.4% |
| **CD_Doppler** | 11.97 | 12.21 | **9.40(最优)** | 10.31 | — |
| CD | 8.08 | 8.34 | 8.69 | 8.90 | (12.23) |
| v_r std | 4.99 | 4.65 | 1.12 | **0.55 ⚠️** | 3.52 |

**三个结论**:

1. **ego 条件是物理一致性的决定性使能**:单加 ego 条件 PCE −89%(8.06→0.92)、CD_Doppler 全场最优(9.40)。验证了轮 1 的诊断——v_r 的自车分量必须由条件解释,不能靠 LiDAR 猜。
2. **物理损失在 ego 条件之上把静态一致性推到极致**:ego_phys 的 PCE(89.3% <0.5)**远超 GT 本身(36.4%)**——生成的静态背景 Doppler 比真实雷达还"干净"(真实雷达含杂波/多径)。约束机制完全生效。
3. **⚠️ 新问题(诚实记录):动态多样性塌缩**。v_r std 一路降到 0.55(GT 3.52)——"比 GT 更一致"实为**红旗**:模型把几乎所有点都生成为静态 Doppler,动态车流的 v_r 多样性被压掉(ego_base 已现端倪 1.12,物理损失加剧)。**PCE 单独解读会误导,必须与 v_r 分布/动态指标配对**(已写进指标解读)。

**成因**:当前只实现了主线 A 的**静态半边**;静态-only 约束下模型学到"全静态最安全"是预期行为 → 补动态分支(见 §4)。

## 4. 消融轮 3:动态一致性 L_dop → **动态多样性显著恢复,主线 A 三组件机制全部验证**

**实现**([losses/physics.py](../code/losses/physics.py) `dynamic_consistency_loss`):利用扩散训练中 **x̂0 与 GT 点 1:1 对应**,对 GT 框标签为运动目标的点监督 `v_r ≈ (v_obj − v_plat(p̂))·r̂(p̂)`(几何取生成点、速度取框速度);判动门限抬至 1.0 m/s 缓解框速度差分噪声(R3,训练集动态点占比 2.9%)。缓存 v3(+label/v_obj);新增分布指标:**W1(v_r)**(池化 1D Wasserstein)与**动态样占比**(|静态残差|>1 的点比例)。

| val N=24 中位 | ego_phys3(静0.1) | **ego_dyn(静0.1+动0.1)** | GT |
|---|---|---|---|
| **动态样占比** | 3.5%(塌缩) | **32.3%(≈10×)** | 57.6% |
| v_r std | 0.62 | **1.12** | 3.52 |
| W1(v_r) | 2.810 | **2.656** | 0 |
| PCE <0.5 | 90.0%(过度) | **38.6%(≈GT 36.4%,健康)** | 36.4% |
| CD / MMD / JSD | **8.99** / 0.064 / 0.390 | 9.57 / 0.112 / 0.450 | (12.23) |

**读数**:
1. **L_dop 方向完全正确**:动态样占比 3.5%→32.3%、v_r std 近翻倍、W1 改善;PCE 从"过度一致(90%)"回到与 GT 混合分布一致的健康区间(38.6%≈36.4%)——生成云现在是"静态严格自洽 + 动态按框速度自洽"的**混合体**,正是主线 A 的目标形态。
2. **代价**:几何指标退化(CD +6%、MMD +75%)——λ_dyn=0.1 对 2.9% 的点权重偏重;这是可调的帕累托,不是机制问题。
3. **注**:GT"动态样占比"57.6% 远高于框标注动态 2.9%——差额是雨天/城市 val 场景的杂波尾巴,与 PCE(GT) med 2.25 一致;该指标衡量的是残差分布尾部形态的匹配,不是真实动态占比。

**至此 proposal 主线 A 的三个组件(ego 条件 + 静态解析硬约束 + 动态一致性软损失)全部实现并在真实数据上验证了各自的预期效应。**
