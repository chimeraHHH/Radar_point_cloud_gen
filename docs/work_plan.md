## 1. 分阶段详细计划

### P0 立项核查 & 环境就绪（W1–2）
- **竞品重扫（最高优先）**：Semantic Scholar / Google Scholar 扫 RadarGen、4D-RaDiff、DoppDrive、LiDARCrafter 的引用网络，确认 2026 上半年无同期「物理约束雷达生成 / 时序雷达生成」竞品。→ 若已有，立即按调研的差异点重定位。
- **代码可得性核查**：4D-RaDiff / RadarGen / RaFlow / LiDARCrafter / DoppDrive 是否开源。⚠️ 两篇核心 prior art 为 2025.12 预印本，**代码大概率未放出**，据此决定基线策略（复现 vs 自建最小基线）。
- **环境**：SSH Key 申请、conda 环境（命名 `xx_radar`）、CUDA 对齐；下载 **VoD** 与 **MAN TruckScenes** 到 `~/data/public_dataset`，NVMe 软链到工作区。
- **协作**：GitHub repo 加 Projects 看板 + issues，确立分工与周会。
- **产出**：竞品核查备忘、环境就绪清单、数据字段/统计报告。
- **G0**：空白仍成立 & 数据可用 & 至少一条基线路径清晰 → 继续；否则重定位课题。

### P1 基线 & 数据管线（W3–5）
- **VoD 管线**：点云读取、坐标系/ego-motion、**静/动标注**（用 3D 框 + 补偿 Doppler）、raw vs 补偿 v_r 对齐。
- **评估脚本**：Chamfer / MMD / JSD + **CD_Doppler / MMD-Doppler**（对标 4D-RaDiff、RadarGen）。
- **基线**：LiDAR→Radar 点（latent）扩散，每点 `(x,y,z,v_r,RCS)`。若无开源 → 自建最小 point-latent-diffusion，文中明确标注「类 4D-RaDiff 重实现」。
- **并行（成员）**：预取 TruckScenes 时序管线 + ego-pose 对齐，为 B 铺路。
- **产出**：可跑基线 + 全套指标 + baseline 数字表。
- **G1**：基线能生成合理点云、Doppler 指标复现到合理量级 → 进入 A。

### P2 主线A：单帧物理一致性（W6–9）
- **静态解析硬约束**：静态点 `v_r ← −v_ego·r̂` 强监督/投影层；消融 vs 无约束、vs 软约束。
- **动态一致性软损失**：`L_dop = |v_r_pred − (v_obj−v_ego)·r̂|`，`v_obj` 取自框/场景运动场；权重与调度策略。
- **反事实速度可控**：干预 `v_ego / v_obj` → 检验生成 `v_r` 的可预测响应（借 arXiv:2605.00018 速度干预测试）。
- **新指标**：多普勒物理一致性误差。**下游**：VoD 检测 mAP（对标 46.0→53.3）、速度估计增益。
- **产出**：主线A 完整实验 + 消融 + 论文 A 段落初稿。
- **G2★**：物理约束是否带来 (a) Doppler 一致性↑ 且 (b) 下游增益↑？
  - **是** → A 可作独立投稿兜底单元；推进 B。
  - **否** → 诊断损失/权重/分割质量；必要时以 A 收缩投稿或调整设计。

### P3 主线B：时序 FlowRadar-4D（W10–14）
- **多帧一致序列生成**骨架（TruckScenes 20Hz）。
- **Doppler 驱动帧间 warp**：用实测/生成 v_r 沿径向推进点（替代 LiDAR 时序工作的注入轨迹）；**首帧 warp 抗漂移** + 保留**纯 ego-warp 稳健下限**对照。
- **双向一致性损失**：`L_temp = |Δrange − v_r·Δt|`（改造 RaFlow `L_rd`）。
- **时序指标**：FVD / FVMD / CTC / TTCE + **速度场一致性** `‖v̂_r·Δt − Δrange‖`。
- **产出**：主线B 完整实验 + demo 序列可视化。
- **G3**：Doppler-warp + 时序损失是否优于 ego-only 下限与「单帧堆叠」？

### P4 整合与写作（W15–18）
- 补消融、跨数据集（VoD↔TruckScenes）、失败案例、限制小节。
- 主图/表/可视化/demo 视频；intro 复用调研三条核心论点。
- 论文写作 + 内部评审 ≥2 轮；预留 rebuttal 素材。**W18 末内部 freeze**。

### P5 缓冲 & 投稿（W19）
- 缓冲期吸收延误；abstract 注册、supp、最终排版与提交（截稿前 2 天完成）。
